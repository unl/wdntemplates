Consider this webpage for a moment and look at the navigation. Is this the navigation for the college, the department, or for UNL? Your guess is as good as mine. We need to associate a breadcrumb element with the current navigation.

If you do nothing when upgrading your templates, the SECOND element within the breadcrumbs will be selected by default. Click here to see what that looks like. This would visually tell the user that the navigation displayed by default is associated with the "College" level. So the "About Us" link could be interpreted by the end-user that the page would have information about the college.

If you would like to specify a different breadcrumb element to be associated with the navigation, you need to tell the templates which element to use. This is done through a <link rel="home" href="" /> tag within the head of each page.

The value of the href attribute is important and must be specified correctly for this association to work. If the navigation is to be associated with the "Department" level of the breadcrumbs, the href should point to the department's homepage. Click here to see what that looks like. This would visually tell the user that the navigation displayed by default is associated with the "Department" level. So the "About Us" link could be interpreted by the end-user that the page would have information about the department.

If you specify this value, make sure you use the exact same address within the breadcrumbs throughout your site. Each of the links below may take the user to the same page, but the addresses are different.

            http://college.unl.edu/department
            http://college.unl.edu/department/
            http://college.unl.edu/department/index.shtml

Consistency is important for many reasons.
